Web Applications

Evil Communicator-Inator

Simple WebRTC client-server pair built using Spring, MongoDB, and Node.

Tech: Java, Spring Boot, Spring Data, Mongo, bcrypt, JS

I chose Java/Spring for this project for its ease of developement and Object Oriented programming, allowing for easy MVC/POJO designs. This would help the modularity of each component and make it seamless to build up new features, additions, or refactoring of the control flow.

MongoDB was my database of choice because of its more loose structuring and natural JSON formatting, meaning that I could easily translate POJOs to and from Mongo without the need for additional code or overheads.
